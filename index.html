Version Control

Version control is a system or method used to track and manage changes to files, documents, or source code over time. It allows multiple people to collaborate on a project by providing a systematic way to track modifications, revert to previous versions, and merge changes made by different contributors. In the context of software development, version control systems (VCS) are commonly used to manage source code files. These systems enable developers to keep track of changes made to code files, maintain a complete history of revisions, and work on different branches of the codebase simultaneously. Version control provides several key benefits, including:

  1. History and Audit Trail:

    It maintains a chronological record of changes, allowing users to review and understand the evolution of a project over time. This history can be valuable for debugging, auditing, or tracking the introduction of new features.

  2. Collaboration:

    Version control systems enable multiple developers to work on the same project simultaneously. They provide mechanisms for merging changes made by different team members, resolving conflicts, and ensuring the stability and consistency of the project.

  3. Branching and Parallel Development:

    Version control allows for the creation of branches, which are independent lines of development. Branches enable developers to work on new features or bug fixes separately without affecting the main codebase. Once the changes are tested and reviewed, they can be merged back into the main branch.

  4. Rollback and Revert:

    If errors or issues are introduced into a project, version control allows for easy rollback to a previous known working state. It provides the ability to revert changes selectively or completely, undoing modifications that may have caused problems.

  5. Collaboration and Code Review:

    Version control systems often provide features for code review, allowing team members to provide feedback, comment on changes, and ensure code quality before merging changes into the main branch.


Commonly used version control systems include Git, Mercurial, and Subversion. These systems offer a range of functionalities and workflows to support different project requirements and team sizes. Overall, version control plays a crucial role in modern software development by providing a structured and efficient way to manage code changes, collaborate effectively, and maintain project integrity over time.

Git and GitHub

Git is a distributed version control system (VCS) widely used in software development. It allows developers to track changes, collaborate with others, and manage source code efficiently. Git is designed to be fast, scalable, and flexible, making it a popular choice for both small and large projects.

Here are some key concepts and features of Git:

  1. Repository: A Git repository is a central storage location for your project's source code and related files. It contains the complete history of changes made to the project.
  2. Commits: A commit in Git represents a snapshot of the project at a specific point in time. It captures the changes made to files and includes a commit message that describes the modifications.
  3. Branching: Git provides a powerful branching model, allowing you to create separate lines of development. Each branch can be used for a specific feature, bug fix, or experiment. Branches make it easy to work on different tasks simultaneously and isolate changes from the main codebase.
  4. Merging: Git enables you to merge changes from one branch to another, combining different lines of development. Merging integrates the changes made in one branch into another, allowing for collaboration and the incorporation of new features or bug fixes.
  5. Remote Repositories: Git supports remote repositories, which enable collaboration and code sharing across different locations. Developers can push their local changes to a remote repository and pull changes made by others.

GitHub, on the other hand, is a web-based hosting service for Git repositories. It provides additional features and functionalities built around Git, including:

  1. Repository Hosting: GitHub allows you to create remote repositories for your projects and provides a web interface to manage and access them. It offers features like repository cloning, browsing, and code review.
  2. Collaboration: GitHub facilitates collaboration by enabling multiple developers to work on the same project simultaneously. It provides tools for managing issues, pull requests, and code reviews.

Git is a version control system that manages and keeps track of your code. GitHub is a service that lets you host, share, and manage your code files on the internet. GitHub uses Git underneath, and lets you manage your Git repositories or folders easily on its platform.

Git allows multiple developers to work together on the same project with ease. You can use Git without GitHub, but you can't use GitHub without Git.

Some of the uses of Git and GitHub include:

  1. Contributing to open source projects
  2. Documentation
  3. Integration options
  4. Tracking changes in your code across versions
  5. Showcasing your work

Using Git and GitHub serves as a backup. Because GitHub has a copy of all of the code you have locally, if anything were to happen to your computer, you’d still have access to your code. Using Git and GitHub allows you to use version control.

Commonly Used Git Commands

  1. Initializes a new Git repository:

    $ git init

  2. Clones a remote repository into a new local directory:

    $ git clone <URL>

  3. Adds a file to the index for preparation for commit:

    $ git add <file>

  4. Commits the changes added to the index with a descriptive message:

    $ git commit -m "message"

  5. Sends local commits to a remote repository:<< /P>$ git push

  6. Retrieves the latest changes from a remote repository and merges them with the local branch.

    $ git pull

  7. Displays the list of available branches and highlights the current branch.

    $ git branch

  8. Switches to another specified branch.

    $ git checkout <branch>

  9. Merges a specified branch with the current branch.

    $ git merge <branch>

  10. Displays the history of commits made in the repository.

    $ git log

  11. This command allows you to set aside uncommitted local changes in a temporary stack, so that you can work on another branch or perform other operations without compromising your ongoing work.

    $ git stash

  12. Used to selectively apply a single commit from one branch to another. This allows you to copy a specific commit from one branch to another, without having to merge the entire branch.

    $ git cherry-pick

  13. Displays a detailed journal of all actions performed in your local repository, including commits, branch changes, resets, etc. This can be useful if you have accidentally lost commits or branches and want to recover them.

    $ git reflog

  14. Displays a detailed journal of all actions performed in your local repository, including commits, branch changes, resets, etc. This can be useful if you have accidentally lost commits or branches and want to recover them.

    $ git reflog

  15. This command displays the current state of your working directory and staging area. It shows which files have been modified, added, or deleted.

    git status:

  16. This command shows the differences between the working directory and the staging area. It helps you review changes before staging them.

    git diff:

  17. This command associates a remote repository with your local repository. The <name> is a short name for the remote repository, and <URL> is the location of the remote repository.

    git remote add <name> <URL>:

  18. This command lists all the remote repositories associated with your local repository, along with their URLs.

    git remote -v:

  19. This command retrieves the latest changes from the remote repository without merging them into your local branch. It updates your remote-tracking branches.

    git fetch:

  20. This command lists all the local and remote branches in your repository.

    git branch -a:

  21. This command deletes the specified branch. The branch must be fully merged into the current branch for it to be deleted.

    git branch -d <branch>:

  22. This command deletes the specified branch from the remote repository.

    git push origin --delete <branch>:

  23. This command undoes the commits after <commit>, discarding the changes in the working directory. The commits are still available in the reflog.

    git reset <commit>:

  24. This command creates a new commit that undoes the changes introduced by the specified commit. It's a safe way to undo commits while preserving history.

    git revert <commit>:

  25. This command creates a lightweight tag at the current commit. Tags are often used to mark important points in history, such as releases or milestones.

    git tag <tagname>:

  26. This command creates an annotated tag at the current commit. Annotated tags store additional information such as the tagger, date, and message.

    git tag -a <tagname> -m "message":

  27. This command is used to perform a binary search in the commit history to find the commit that introduced a specific bug. It guides you automatically by asking you whether each commit is good or bad until it finds the problematic commit.

    $ git bisect


GitHub Essentials

GitHub is a web-based platform that uses Git for version control and collaboration. It allows developers to store and manage their code repositories in the cloud. GitHub provides a user-friendly interface that makes it easy for individuals and teams to use Git for version control and collaboration. It also provides access control, bug tracking, software feature requests, task management, continuous integration and deployment, and other features that help developers work together more efficiently

GitHub Enterprise is a self-hosted version of GitHub, which is a web-based platform for version control and collaboration. GitHub Enterprise is designed for organizations that prefer to have their own dedicated instance of GitHub installed and managed on their own infrastructure, rather than using the public GitHub service. GitHub Enterprise offers many of the same features as the public GitHub, including repository hosting, issue tracking, pull requests, code reviews, and collaboration tools. It provides a centralized platform for teams to manage and collaborate on software development projects, allowing multiple developers to work on the same codebase simultaneously. Some key features and benefits of GitHub Enterprise include:

  1. Self-hosted:

    Organizations have full control over the infrastructure and data associated with their GitHub instance. It can be installed on-premises or on a private cloud environment.

  2. Security and compliance:

    GitHub Enterprise offers enhanced security features, such as enterprise authentication options, granular access controls, and audit logs. It allows organizations to meet their specific security and compliance requirements.

  3. Scalability:

    GitHub Enterprise is designed to support large-scale enterprise environments with high-performance requirements. It can handle a large number of repositories, users, and concurrent operations.

  4. Integration and extensibility:

    It integrates with a wide range of development tools and services, allowing seamless workflows and automated processes. It supports integrations with issue trackers, CI/CD pipelines, project management tools, and more.

  5. Support and services:

    GitHub Enterprise comes with enterprise-level support and services, including technical support, training, and consulting, to assist organizations in deploying and managing their GitHub instance effectively.

Overall, GitHub Enterprise offers organizations a private and customizable version control and collaboration platform, tailored to their specific needs and requirements. It provides the benefits of the popular GitHub platform while allowing organizations to maintain control over their codebase and infrastructure.

Download Git for Windows

1. Browse to the official Git website: https://git-scm.com/downloads
2. Click the download link for Windows and allow the download to complete.

Extract and Launch Git Installer

3. Browse to the download location (or use the download shortcut in your browser). Double-click the file to extract and launch the installer.

4. Allow the app to make changes to your device by clicking Yes on the User Account Control dialog that opens.

5. Review the GNU General Public License , and when you’re ready to install, click Next .

6. The installer will ask you for an installation location. Leave the default, unless you have reason to change it, and click Next .

7. A component selection screen will appear. Leave the defaults unless you have a specific need to change them and click Next .

8. The installer will offer to create a start menu folder. Simply click Next .

9. Select a text editor you’d like to use with Git. Use the drop-down menu to select Notepad++ (or whichever text editor you prefer) and click Next .

10. The next step allows you to choose a different name for your initial branch. The default is 'master.' Unless you're working in a team that requires a different name, leave the default option and click Next.

11. This installation step allows you to change the PATH environment . The PATH is the default set of directories included when you run a command from the command line. Leave this on the middle (recommended) selection and click Next .

Server Certificates, Line Endings and Terminal Emulators

12. The installer now asks which SSH client you want Git to use. Git already comes with its own SSH client, so if you don't need a specific one, leave the default option and click Next.

Note: Check out our comparison of SSH and HTTPS for Git and which one you should use.

13. The next option relates to server certificates. Most users should use the default. If you’re working in an Active Directory environment, you may need to switch to Windows Store certificates. Click Next .

14. The next selection converts line endings. It is recommended that you leave the default selection. This relates to the way data is formatted and changing this option may cause problems. Click Next .

15. Choose the terminal emulator you want to use. The default MinTTY is recommended, for its features. Click Next .

16. The installer now asks what the git pull command should do. The default option is recommended unless you specifically need to change its behavior. Click Next to continue with the installation.

17. Next you should choose which credential helper to use. Git uses credential helpers to fetch or save credentials. Leave the default option as it is the most stable one, and click Next .

Additional Customization Options

18. The default options are recommended, however this step allows you to decide which extra option you would like to enable. If you use symbolic links, which are like shortcuts for the command line, tick the box. Click Next .

19. Depending on the version of Git you’re installing, it may offer to install experimental features. At the time this article was written, the options to include support for pseudo controls and a built-in file system monitor were offered. Unless you are feeling adventurous, leave them unchecked and click Install .

Complete Git Installation Process

20. Once the installation is complete, tick the boxes to view the Release Notes or Launch Git Bash, then click Finish .
